## 2. Постановка задачи и контекст

### 2.1. Предметная область и задача

Задача — классификация изображений ландшафтов. Потенциальные пользователи: сервисы автоматической разметки фотографий, приложения для туристов, системы мониторинга окружающей среды. Сценарий: пользователь загружает фотографию природы/города, сервис определяет тип ландшафта.

### 2.2. Формулировка задачи в терминах ML

- **Входные данные:** изображение RGB (150×150 пикселей, 3 канала)
- **Целевая переменная:** один из 6 классов (buildings, forest, glacier, mountain, sea, street)
- **Выход модели:** предсказанный класс + вероятности по всем классам
- **Ограничения:**
  - Скорость инференса: < 200 мс на CPU
  - Точность: accuracy > 90% на тестовой выборке
  - Интерпретируемость: модель должна быть объяснимой (ResNet — стандартная архитектура, не black-box)

### 2.3. Целевые метрики

- **Accuracy** — основная метрика, т.к. классы сбалансированы (~15% разницы между минимальным и максимальным классом)
- **Confusion matrix** — для анализа ошибок между похожими классами (glacier/mountain, buildings/street)
- **Precision/Recall по классам** — чтобы убедиться, что нет сильного перекоса в предсказаниях

## 3. Данные

### 3.1. Источник данных

Открытый датасет **Intel Image Classification** с Kaggle: https://www.kaggle.com/datasets/puneet6060/intel-image-classification

### 3.2. Структура данных

- **Train:** ~14,034 изображений, 6 папок по классам
- **Test:** ~3,000 изображений, аналогичная структура
- **Prediction:** ~7,000 изображений без меток (не использовались)
- **Размер изображений:** преимущественно 150×150 px, но встречаются и другие (например, 150×105)
- **Формат:** JPEG

### 3.3. Предобработка и EDA

Основные шаги EDA выполнены в `notebooks/01_eda.ipynb`:

1. **Распределение классов:**
   - Train: buildings(2191), forest(2271), glacier(2404), mountain(2512), sea(2274), street(2382)
   - Test: buildings(437), forest(474), glacier(553), mountain(525), sea(510), street(501)
   - Вывод: классы примерно сбалансированы, разница не критична

2. **Размеры изображений:**
   - Большинство 150×150, но есть и другие
   - Решение: resize до 224×224 при загрузке (требование ResNet)

3. **Цветовой анализ:**
   - forest — преобладание зелёного (низкая общая яркость)
   - sea — высокий уровень синего
   - glacier — самая высокая яркость (белый/голубой)
   - street — тусклые тона, преобладание серого

4. **Аугментации:**
   - RandomHorizontalFlip(p=0.5) — ландшафты инвариантны к отражению
   - RandomRotation(15°) — имитация разных ракурсов
   - ColorJitter(brightness=0.2, contrast=0.2, saturation=0.2) — разные условия освещения

## 4. Модели и подходы

### 4.1. Baseline модели

**Simple CNN** (обучение с нуля):
- Архитектура: 4 свёрточных слоя (32→64→128→256) + 2 FC слоя
- Результат: ~65-70% accuracy на тесте
- Вывод: недостаточно данных для обучения с нуля, сильное переобучение

### 4.2. Улучшенные модели

**ResNet18 Frozen** (transfer learning):
- Замороженный backbone, обучается только классификатор
- Результат: ~88-90% accuracy за 5-7 эпох
- Вывод: transfer learning эффективен, быстрая сходимость

**ResNet18 Fine-tuned** (двухэтапное обучение):
- Этап 1: 5 эпох с замороженным backbone, lr=0.001
- Этап 2: 10 эпох с размороженной сетью, lr=0.0001
- Результат: ~92-94% accuracy
- Вывод: fine-tuning улучшает адаптацию под домен

### 4.3. Нейросетевые модели

- **Архитектура:** ResNet18 (pretrained on ImageNet)
- **Оптимизатор:** Adam
- **Loss:** CrossEntropyLoss
- **Scheduler:** StepLR (step=7, gamma=0.1 для frozen; step=3, gamma=0.5 для fine-tuning)
- **Batch size:** 32
- **Размер входа:** 224×224 (resize из 150×150)

Все эксперименты в `notebooks/02_experiments.ipynb`.

## 5. Экспериментальный протокол и результаты

### 5.1. Протокол

- **Разделение:** фиксированное train/test из датасета (нет отдельного val, используем test для контроля)
- **Метрики:** считаем на test после каждой эпохи
- **Сохранение:** лучшая модель по test accuracy

### 5.2. Сравнение моделей

| Модель | Описание | Test Accuracy | Комментарий |
|--------|----------|---------------|-------------|
| Simple CNN | 4 conv слоя, обучение с нуля | ~68% | Baseline, сильное переобучение |
| ResNet18 Frozen | Transfer learning, замороженный backbone | ~89% | Быстрая сходимость, хороший старт |
| ResNet18 Fine-tuned | Fine-tuning всей сети | ~93% | **Финальная модель** |

### 5.3. Выбор финальной модели

**ResNet18 Fine-tuned** выбрана как финальная:
- Лучшая accuracy (~93%)
- Разумный trade-off между точностью и скоростью (ResNet18 — лёгкая модель)
- Стабильное обучение без резких скачков loss

## 6. Архитектура решения и сервис


### 6.1. Архитектура пайплайна

```
Пользователь → [FastAPI /predict] → [Загрузка изображения]
                                              ↓
[Предобработка: Resize 224×224 + Normalize] → [ResNet18]
                                              ↓
[Softmax] → [Класс + вероятности] → [JSON ответ]
```

Связь компонентов:
- **Данные:** `src/data/dataset.py` — загрузка, аугментации, нормализация
- **Модель:** `src/models/classifier.py` — ResNet18 + классификатор
- **Обучение:** `src/training/train.py` — 2 этапа (freeze → unfreeze)
- **Инференс:** `src/inference/predictor.py` — загрузка весов, предсказание
- **Сервис:** `src/service/api.py` — FastAPI, эндпоинты

### 6.2. API и endpoints

FastAPI сервис (`src/service/api.py`):

| Endpoint | Метод | Описание |
|----------|-------|----------|
| `/health` | GET | Статус сервиса и модели |
| `/predict` | POST | Загрузка одного изображения (multipart/form-data) |
| `/predict_batch` | POST | Батчевая обработка (до 10 изображений) |

**`/predict`:**
- **Принимает:** файл изображения (`file: UploadFile`)
- **Возвращает:**
  ```json
  {
    "predicted_class": "mountain",
    "confidence": 0.9432,
    "probabilities": {
      "buildings": 0.0012,
      "forest": 0.0087,
      "glacier": 0.0234,
      "mountain": 0.9432,
      "sea": 0.0123,
      "street": 0.0112
    }
  }
  ```

### 6.3. Технологический стек

- **ML:** PyTorch, torchvision
- **API:** FastAPI, uvicorn
- **Тесты:** pytest, httpx
- **Конфиги:** PyYAML
- **Контейнеризация:** Docker
