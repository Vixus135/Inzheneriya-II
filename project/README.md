# Итоговый проект по курсу «Инженерия Искусственного Интеллекта»

## 1. Паспорт проекта

- **Название проекта:** `Сервис классификации ландшафтов по изображениям (Intel Image Classification)`
- **Автор:** `Андреева Виктория Дмитриевна`
- **Группа:** `КВБО-08-23>`
- **Контакт:** `andreevavika0506@mail.ru`
- **Ссылка на репозиторий:** `https://github.com/Vixus135/Inzheneriya-II/tree/main`
- **Краткое описание:** Проект посвящён построению сервиса классификации изображений ландшафтов на 6 классов (лес, ледник, горы, море, улица, здания). Используется transfer learning на базе ResNet18 с fine-tuning. Результат - REST API, принимающий изображение и возвращающий предсказанный класс с вероятностями

---

## 2. Структура проекта

Проект организован в следующей структуре:

- `requirements.txt` – зависимости проекта (библиотеки Python, необходимые для запуска).
- `report.md` – отчёт по проекту (постановка задачи, данные, эксперименты, результаты).
- `self-checklist.md` – чеклист самопроверки проекта 
- `notebooks/` – EDA и эксперименты;
- `src/` – основной код (данные, модели, обучение, сервис);
- `data/` – демонстрационные данные (инструкция по загрузке);
- `configs/` – YAML-конфиги;
- `tests/` – pytest-тесты.
- `artifacts/` – сохранённые модели, отчёты, результаты экспериментов.

## 5. Данные

Используется открытый датасет **Intel Image Classification** (25K изображений, 6 классов).
- **Источник:** [Kaggle](https://www.kaggle.com/datasets/puneet6060/intel-image-classification)
- **Train:** ~14,034 изображений (150×150 px)
- **Test:** ~3,000 изображений
- **Классы:** buildings, forest, glacier, mountain, sea, street

## 6. Тесты (если есть)

```bash
cd project
source .venv/bin/activate
pytest tests/
```

Тесты покрывают:
- `test_health_check` - проверка health эндпоинта
- `test_predict_valid_image` - предсказание на валидном изображении
- `test_predict_wrong_file_type` - отклонение не-изображений (400)
- `test_predict_no_file` - ошибка при отсутствии файла (422)
- `test_batch_predict_two_images` - батчевая обработка
- `test_batch_predict_too_many` - ограничение на 10 файлов (400)
- `test_predict_response_types` - проверка типов данных в ответе

## 8. Ограничения и дальнейшая работа

**Текущие ограничения:**
- Только 6 классов ландшафтов (узкий домен)
- Нет кэширования результатов (каждый запрос пересчитывается)
- Нет авторизации API (открытый доступ)
- Нет логирования запросов
