# S03 – eda_cli: мини-EDA для CSV

Небольшое CLI-приложение для базового анализа CSV-файлов.
Используется в рамках Семинара 03 курса «Инженерия ИИ».

## Требования

- Python 3.11+
- [uv](https://docs.astral.sh/uv/) установлен в систему

## Инициализация проекта

В корне проекта (S03):

```bash
uv sync
```

Эта команда:

- создаст виртуальное окружение `.venv`;
- установит зависимости из `pyproject.toml`;
- установит сам проект `eda-cli` в окружение.

## Запуск CLI

### Краткий обзор

```bash
uv run eda-cli overview data/example.csv
```

Параметры:

- `--sep` – разделитель (по умолчанию `,`);
- `--encoding` – кодировка (по умолчанию `utf-8`).

### Полный EDA-отчёт

```bash
uv run eda-cli report data/example.csv --out-dir reports
```

В результате в каталоге `reports/` появятся:

- `report.md` – основной отчёт в Markdown;
- `summary.csv` – таблица по колонкам;
- `missing.csv` – пропуски по колонкам;
- `correlation.csv` – корреляционная матрица (если есть числовые признаки);
- `top_categories/*.csv` – top-k категорий по строковым признакам;
- `hist_*.png` – гистограммы числовых колонок;
- `missing_matrix.png` – визуализация пропусков;
- `correlation_heatmap.png` – тепловая карта корреляций.
- `category_bar_<column>.png` – bar-chart по выбранной категориальной колонке (дополнительная визуализация).

### Новые параметры команды report
- `--max-hist-columns` — сколько числовых колонок включать в гистограммы (по умолчанию 6).

- `--top-k-categories` — сколько top‑значений выводить для категориальных признаков (по умолчанию 5).

- `--title` — заголовок отчёта (по умолчанию EDA-отчёт).

- `--min-missing-share` — порог доли пропусков, выше которого колонка считается проблемной и попадает в отдельный список.

Все параметры отражаются в итоговом report.md.

### Эвристики качества данных
В отчёте проверяются дополнительные признаки:

- `has_constant_columns` — наличие константных колонок;

- `has_suspicious_id_duplicates` — дубликаты в колонках, похожих на идентификаторы;

- `has_many_zero_values` — числовые колонки с большим количеством нулей;

- список проблемных колонок по порогу `min_missing_share`.

### Дополнительная визуализация
В модуле `viz.py` добавлена функция `plot_simple_category_bar`, которая строит bar‑chart по количеству объектов в выбранной категориальной колонке. График сохраняется в PNG и упоминается в `report.md`

## Тесты

```bash
uv run pytest -q
```

## Пример запуска
```bash
uv run eda-cli report data/example.csv \
    --out-dir reports_example \
    --max-hist-columns 4 \
    --top-k-categories 10 \
    --title "Отчёт по Example" \
    --min-missing-share 0.1
```
После выполнения команда создаст каталог `reports_example` с артефактами и файлом `report.md`.

## Структура артефактов
- `report.md` — основной отчёт.
- `summary.csv` — сводка по колонкам.
- `missing.csv`, `missing_matrix.png` — статистика и визуализация пропусков.
- `correlation.csv`, `correlation_heatmap.png` — корреляция числовых признаков.
- `top_categories/*.csv` — top‑значения категориальных признаков.
- `hist_*.png` — гистограммы числовых колонок.
- `category_bar_city.png` — bar‑chart по выбранной категориальной колонке.
